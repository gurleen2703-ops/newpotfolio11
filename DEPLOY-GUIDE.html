Part 1

First Time Setup

You only do this once. Takes about 10 minutes total.

1

Create a GitHub Account

Go to github.com → Click "Sign Up" → Enter your email, create a password, choose a username → Verify your email.

💡 Use your business email for a professional look. Your GitHub username will appear in your repo URL.
2

Create a New Repository

After logging in to GitHub:

① Click the green "New" button or the + icon → "New repository"
② Repository name: wss
③ Set it to Public (required for free Vercel hosting)
④ Leave everything else as default → Click "Create repository"

3

Install Git on Your Computer

Git is the tool that sends your files to GitHub.

→ Windows: Download from git-scm.com → Install with all default settings
→ Mac: Open Terminal → type git --version → if not installed, it will prompt you to install it

After installing, open a terminal (VS Code terminal works) and configure your identity:

$git config --global user.name "Gurleen Kaur"
$git config --global user.email "worldsocialsolutions6@gmail.com"
4

Create a Vercel Account

Go to vercel.com → Click "Sign Up" → Choose "Continue with GitHub" → Authorize Vercel.

💡 Signing up with GitHub links both accounts automatically — Vercel can then see and deploy your repos directly.
Part 2 — Method A

Deploy via Drag & Drop

Quickest way to get your site live. No GitHub required.

1

Go to Vercel Drop

Open vercel.com/new in your browser. Log in if prompted.

2

Drag Your Folder

Find your wss folder on your computer. Drag the entire folder onto the Vercel webpage where it says "Import Project".

⚠ Important: Drag the wss folder itself — not just the files inside it. The folder should contain index.html, pages/, admin/, etc.
3

Deploy

Vercel detects it's a static site. Leave all settings as default → Click "Deploy".

In about 30 seconds you'll get a URL like: wss-abc123.vercel.app 🎉

💡 Your admin panel is also live at: wss-abc123.vercel.app/admin/
⚠ Limitation: With drag & drop, you have to re-drag every time you make changes. For ongoing updates, use Method B below.
Part 2 — Method B (Recommended)

Deploy via GitHub + Vercel

Set up once — then every change you push to GitHub automatically goes live on your site.

1

Open Your Project in VS Code Terminal

In VS Code, open the wss folder. Then open the terminal:

Terminal → New Terminal (or press Ctrl + `)

Make sure the terminal shows you're inside the wss folder.

2

Initialise Git & Push to GitHub

Run these commands one by one in the terminal:

# Initialise git in your project
$git init
# Stage all files
$git add .
# Create first commit
$git commit -m "initial website launch"
# Set main branch name
$git branch -M main
# Connect to your GitHub repo (replace YOUR_USERNAME)
$git remote add origin https://github.com/YOUR_USERNAME/wss.git
# Push files to GitHub
$git push -u origin main
💡 Replace YOUR_USERNAME with your actual GitHub username. You can find the exact URL on your GitHub repo page — click the green "Code" button → copy the HTTPS URL.
3

Import to Vercel

① Go to vercel.com/new
② Click "Import Git Repository"
③ You'll see your wss repo listed → Click "Import"
④ Framework Preset: leave as "Other"
⑤ Root Directory: leave as "./" (default)
⑥ Click "Deploy"

4

Your Site is Live! ✅

Vercel gives you a URL like wss.vercel.app or wss-yourusername.vercel.app

→ Main website: https://wss.vercel.app/
→ Admin panel: https://wss.vercel.app/admin/

✅ From now on, every time you push to GitHub, Vercel auto-deploys your site in ~20 seconds. No manual steps needed.
Part 3 — Most Important

How to Update Live Site Content

This is the workflow you'll use every time you want to update your website after it's live.

ℹ️ Why is this needed? The Admin Panel saves to your browser's localStorage — which only exists on your computer. To make changes appear on the live Vercel site, you need to export and push the updated data file.
1

Open Admin Panel Locally

In VS Code, right-click index.html → "Open with Live Server" → Navigate to http://127.0.0.1:5500/admin/

2

Make Your Changes

Add a new portfolio item, update a testimonial, change your phone number — whatever you need. Click the Save button for each section.

3

Click "⬇ Export sitedata.js"

In the Admin Panel, click the "⬇ Export sitedata.js" button in the top header or the sidebar. This downloads a file called sitedata.js to your Downloads folder.

💡 This file contains ALL your content baked in — so the live site will use it directly without needing localStorage.
4

Replace the File in Your Project

Copy the downloaded sitedata.js file and paste it into your project's data/ folder, replacing the existing one.

Path: wss/data/sitedata.js

5

Push to GitHub

In the VS Code terminal:

$git add data/sitedata.js
$git commit -m "update site content"
$git push
6

Vercel Deploys Automatically ✅

Vercel detects the push and deploys in ~20 seconds. Visit your Vercel dashboard to watch the deployment. Your live site is updated!

Part 4

Add Your Custom Domain

Connect worldsocialsolutions.in (or any domain) to your Vercel project.

1

Buy a Domain (if you haven't already)

Popular registrars for Indian domains:
→ GoDaddy.com — .in domains from ~₹800/year
→ BigRock.in — good Indian registrar
→ Hostinger.in — affordable options

2

Add Domain in Vercel

① Go to vercel.com/dashboard
② Click your wss project
③ Go to Settings → Domains
④ Type your domain: worldsocialsolutions.in
⑤ Click Add

Vercel will show you two DNS records to add — keep this tab open.

3

Add DNS Records at Your Registrar

Log in to your domain registrar (GoDaddy, BigRock, etc.) → Find DNS Management or DNS Settings.

Add the records Vercel showed you. Usually:

TypeNameValue
A@76.76.21.21
CNAMEwwwcname.vercel-dns.com
💡 The exact values will be shown by Vercel — copy them from there, don't type from memory.
4

Wait for DNS Propagation

DNS changes take 5 minutes to 48 hours to propagate globally. Usually it's under 30 minutes.

Once done, your site is live at https://worldsocialsolutions.in with a free SSL certificate (HTTPS) included automatically by Vercel. ✅

Part 5

Troubleshooting

Common issues and how to fix them.

Problem Fix
Admin changes don't show on live site Export sitedata.js → replace data/sitedata.js → push to GitHub
Git push asks for username/password Use a GitHub Personal Access Token — GitHub Settings → Developer Settings → Tokens → Generate
Vercel build fails Check that index.html is in the root of the wss folder. Framework must be set to "Other"
Images not showing on live site Make sure image files are inside assets/images/ and paths in admin are correct (case-sensitive on Linux)
Domain not connecting DNS can take up to 48 hours. Use dnschecker.org to check propagation
git command not found Install Git from git-scm.com and restart VS Code
Pages give 404 on Vercel Add a vercel.json file to your project root (see below)

If pages give 404 errors, create a file vercel.json in your wss folder with this content:

{
"rewrites": [{ "source": "/(.*)", "destination": "/$1" }]
}

Commands Cheat Sheet

$git status
